# 目录当前章节定位修复设计

## 问题

经典目录与分卷目录的 `LazyColumn` 都未设置初始滚动索引。打开目录时列表默认从索引 0 开始，因此当前章节虽能高亮，但用户必须手动滚动才能看到。

## 设计

- 在 `ReaderContentsCatalog` 增加纯函数，分别计算经典章节列表和分卷可见条目列表中当前章节的位置。
- 经典目录用当前章节的真实 `chapter.index` 查找列表位置，避免搜索过滤后把列表位置误当章节索引。
- 分卷目录先展开当前章节所在分组，再从 `visibleItems` 中查找当前章节条目位置。
- 两种目录都使用计算出的索引初始化 `LazyListState`，首次绘制直接显示当前章节，避免先显示第一章再跳转。
- 当前章节不存在或列表为空时安全回退到索引 0。
- 用户输入目录搜索词后不再自动改变滚动位置，避免搜索过程中抢夺用户滚动。

## 验证

- 单元测试覆盖经典列表、过滤列表、分卷列表和当前章节缺失时的索引计算。
- 运行目录相关单元测试、全部 App 单元测试及 Debug Kotlin 编译。
