# 汗牛充栋 · 阅读书籍管理系统

## 一、项目概述

**项目名称**：汗牛充栋

**项目定位**：面向个人及家庭用户的私有化图书管理与阅读平台，部署于家庭局域网 NAS 设备，支持本地书籍管理、网络书籍爬取、多端在线阅读，以及主流开源客户端协议接入。

**核心目标**：构建一个"买书、存书、读书"一站式私有书库，数据完全自持，无依赖外部订阅服务。

---

## 二、技术架构

### 2.1 技术栈

| 层次 | 技术选型 |
|---|---|
| 前端 | Vue 3 + Vite + Pinia + Vue Router |
| UI 组件 | Element Plus 或 Naive UI |
| 后端 | Java 21 + Spring Boot 3 + Spring Security |
| 数据库 | PostgreSQL 16 |
| 搜索索引 | PostgreSQL 全文检索（可选 Meilisearch） |
| 任务调度 | Spring Scheduler / Quartz |
| 容器化 | Docker + Docker Compose |
| CI/CD | Jenkins（NAS 本地部署） |
| 代码托管 | GitHub |

### 2.2 开发环境

- macOS，所有服务通过 Docker Compose 在本地运行
- 资源文件（封面图、书籍原文件）放置于项目根目录 `resources/` 下
- 大文件（> 50MB）通过 `.gitignore` 排除，仅保留元数据和配置

### 2.3 生产环境

- 飞牛 NAS 上的 Docker 服务
- Jenkins 监听 GitHub 主分支，触发自动构建和容器更新
- 书籍文件挂载为 Docker Volume，持久化存储

## 三、功能模块详细需求

### 3.1 本地书库管理

#### 文件夹扫描

- 支持配置**多个监控目录**（如 `/books/fiction`, `/books/tech`）
- 支持**手动扫描**和**定时自动扫描**（如每日凌晨）
- 扫描时进行去重检测，基于文件哈希（MD5/SHA256）避免重复入库
- 扫描结果展示：新增 N 本、更新 N 本、失败 N 本，失败原因可查

#### 支持的书籍格式

| 格式 | 说明 |
|---|---|
| `.txt` | 纯文本，支持编码自动检测（UTF-8 / GBK） |
| `.epub` | 主流电子书格式，支持目录解析 |
| `.mobi` / `.azw3` | Kindle 格式 |
| `.pdf` | 支持文本层提取，纯扫描版标记为图片 PDF |
| `.docx` / `.doc` | Word 文档 |
| `.html` / `.htm` | 网页存档 |
| `.cbz` / `.cbr` | 漫画格式（可选支持） |
| `.md` | Markdown 格式 |

#### 元数据管理

- 自动从文件内部提取元数据（epub 的 OPF 信息、PDF 的 XMP 数据等）
- 支持对接**豆瓣读书 / Open Library / Google Books API** 自动补全书名、作者、封面、简介、ISBN
- 支持手动编辑所有元数据字段
- 封面图本地缓存，支持手动上传替换

---

### 3.2 网络书籍爬取

#### 书源管理

- 内置常见公开书源规则（仿 BookSource 格式，JSON 配置）
- 支持用户**自定义书源规则**，配置书源站点的搜索、目录、正文 XPath/CSS 选择器
- 书源可导入/导出，支持社区共享

#### 搜索与下载

- 全局书名/作者搜索，聚合多个书源结果
- 预览书籍简介和目录后再决定下载
- 支持**全本一次性下载**和**增量追更**（连载中作品定时同步最新章节）
- 下载任务队列管理，支持暂停、继续、取消
- 自动将爬取内容转换为 epub 格式存档

#### 反爬与合规

- 请求间隔可配置，默认遵守 robots.txt
- 用户代理伪装，支持代理 IP 配置
- 明确标注：本功能仅用于下载有权阅读的免费内容，版权责任由用户自行承担

---

### 3.3 书籍查看与阅读

#### 3.3.1 书籍列表管理

- 分页展示，支持卡片视图（封面优先）和列表视图（密度高）
- 多维筛选：格式、语言、标签、评分、添加时间、阅读状态
- 多字段排序：书名、作者、添加时间、最近阅读、评分
- 全文搜索：书名、作者、ISBN、简介
- 批量操作：批量打标签、批量移动分类、批量删除

#### 3.3.2 我的书架

| 功能 | 说明 |
|---|---|
| 收藏 | 标记喜爱书籍，快速访问 |
| 正在阅读 | 当前进度跟踪，继续阅读入口 |
| 已读完 | 归档，支持添加读后评分和笔记 |
| 想读 | 待读清单 |
| 自定义书单 | 用户创建命名书单，如"2025年读书计划" |
| 阅读统计 | 年/月阅读量、累计时长、完成率图表 |

#### 3.3.3 网页在线阅读器

- **txt / epub / html / md**：基于分页或滚动的文本阅读器
  - 字体选择（衬线/无衬线）、字号、行间距、背景色（白/米/黑）
  - 章节目录侧边栏，支持书签
  - 阅读进度自动保存（精确到章节 + 百分比）
  - 划词高亮 + 批注功能
- **PDF**：基于 PDF.js 的 PDF 阅读器，支持双页模式、缩放
- **docx**：后端转换为 HTML 后在浏览器渲染

#### 3.3.4 移动端与第三方客户端支持

支持以下开放协议，允许 KOReader、Moon+ Reader、Librera 等客户端直接连接：

| 协议 | 用途 |
|---|---|
| **OPDS 1.2 / 2.0** | 书目目录浏览与下载（兼容绝大多数开源阅读器） |
| **WebDAV** | 文件同步（KOReader 进度同步、直接下载） |
| **KOReader 书架同步** | 阅读进度、高亮、笔记的双向同步 |

---

### 3.4 系统管理

- **用户管理**：支持多用户，各用户独立书架和阅读进度（家庭共用场景）
- **存储管理**：磁盘使用量监控，文件清理工具（删除无元数据的孤立文件）
- **任务中心**：查看扫描、爬取、转换任务的实时状态和历史日志
- **系统设置**：扫描目录配置、书源管理、定时任务配置、API Token 管理

## 四、非功能需求

- **性能**：书库 10,000 本以内列表响应 < 500ms；大文件（> 100MB PDF）流式传输，首字节 < 1s
- **安全**：JWT 鉴权，局域网部署可选关闭登录；OPDS 支持 Basic Auth
- **可维护性**：Docker Compose 一键启动，Jenkins Pipeline 脚本化部署，配置通过环境变量注入
- **数据安全**：数据库定期自动备份到本地另一目录，支持手动导出 JSON 格式元数据
