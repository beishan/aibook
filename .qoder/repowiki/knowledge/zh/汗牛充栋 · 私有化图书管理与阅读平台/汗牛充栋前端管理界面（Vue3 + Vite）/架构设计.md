入口 `src/main.ts` 通过 `createApp` 挂载 App、Pinia、vue-router 与 ElementPlus，并调用 `useThemeStore().initTheme()` 初始化主题。
- 路由层：`src/router/index.ts` 使用 `createWebHistory`，以 `/` 为根嵌套在 `ThemeLayoutWrapper.vue` 下，子路由按业务划分到 `views/`；通过 `meta.requiresAuth` 配合 `beforeEach` 守卫实现基于 `localStorage.token` 的鉴权跳转。
- 状态层：`src/stores/` 下每个领域一个 Pinia store（user、book、category、tag、theme），采用 Composition API 风格 `defineStore('id', () => {...})`，内部直接调用 `@/utils/api` 发起请求并同步持久化到 localStorage。
- 网络层：`src/utils/api.ts` 封装 axios 实例，统一注入 Bearer token、超时与全局错误处理（401 自动跳回 /login，其余用 ElMessage 提示）。
- 视图层：`src/views/` 对应页面级组件，布局由 `src/layouts/` 下的 Dock/Sidebar/Topbar 三种 Layout 组合 `ThemeLayoutWrapper` 提供。
- 可复用 UI：`src/components/` 存放跨页面组件（ScraperDialog、BatchScraperDialog、DirectoryBrowser、FileUpload、Highlights、MetadataSearch、Bookmarks、TreeNode）。
- 构建与代理：`vite.config.ts` 配置 `@` 别名指向 `src`，开发服务器端口 3000，并将 `/api` 反向代理至后端 `http://localhost:8080`；生产产物输出到 `dist/`，由同目录 `nginx.conf` 与 `Dockerfile` 托管。
