根目录作为跨子模块的编排层：
- 运行期由 `docker-compose.yml`（位于 docker/）统一拉起 backend_service、frontend_admin、android_app 以及 PostgreSQL 16、Redis、MinIO 等基础设施容器，共享 `.env` 中的 DB_PASSWORD/REDIS_PASSWORD/MINIO_PASSWORD/JWT_SECRET 等凭据。
- 前后端通过 REST API 解耦：frontend_admin 调用后端 `/api/*` 进行书籍元数据与用户管理；android_app 除 REST 外还消费后端暴露的 OPDS 1.2/2.0 与 WebDAV 目录以兼容 KOReader/Moon+ Reader 等第三方客户端。
- 持久化与对象存储为共享边界：所有子模块共用同一 PostgreSQL 库（books/book_files/users/reading_progress/book_lists 表族）与 MinIO 桶（存放书籍原文件与封面），确保多端数据一致。
- 根级 AGENTS.md / CLAUDE.md / requires_1.md 作为需求与开发规范入口，约束各子工程的代码风格（Java Google Style、Vue Composition API + `<script setup>`）、Git 提交格式与分支策略。
