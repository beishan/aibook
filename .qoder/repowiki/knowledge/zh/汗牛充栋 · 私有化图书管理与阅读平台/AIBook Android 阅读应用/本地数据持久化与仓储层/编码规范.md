- Repository 方法统一返回 `kotlinx.coroutines.flow.Flow` 用于实时观察，或 `suspend` 函数用于一次性写入/查询。
- Entity 与 Domain 之间的转换集中在 `mapper/EntityMappers.kt`，通过扩展函数 `toDomain()` / `toEntity()` 成对定义，避免在 Repository 中散落映射逻辑。
- 从字符串反序列化枚举时使用 `runCatching { X.valueOf(name) }.getOrDefault(DEFAULT)` 模式，保证旧数据迁移时的向后兼容。
- 需要唯一标识的实体（如 OPDS 连接）优先用 `UUID.nameUUIDFromBytes(...)` 基于稳定字段派生 id，而非随机 UUID。
