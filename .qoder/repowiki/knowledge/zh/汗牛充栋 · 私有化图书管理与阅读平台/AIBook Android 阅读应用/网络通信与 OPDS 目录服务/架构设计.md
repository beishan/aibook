模块按功能划分为两个并列子包：
- `api`：面向后端 REST 接口，通过 `ApiServiceFactory` 单例集中装配 `OkHttpClient`（连接/读写超时、`AuthInterceptor` 注入 Bearer Token、可选 `HttpLoggingInterceptor`）、`Retrofit`（`kotlinx-serialization` JSON 转换器）并暴露 `createXxxApi` 工厂方法；各业务域以 Retrofit 接口形式声明（`AuthApi`、`BookApi`、`ReadingProgressApi`），请求体/响应体 DTO 集中在 `dto/` 下。
- `opds`：独立于 Retrofit 的 OPDS 协议实现，采用依赖倒置——`OpdsCatalogService` 组合 `OpdsTransport` 接口与 `OpdsFeedParser`，默认 `OkHttpOpdsTransport` 使用原生 OkHttp 发起 Atom/OPDS XML 请求，`OpdsFeedParser` 通过 JAXP DOM 解析 feed 并映射到 `OpdsModels.kt` 中的数据类；`OpdsRequestFactory` 负责 URL 拼接与 Basic Auth Header 生成。
依赖方向单向：`opds` 不依赖 `api`，两者都只依赖 `core:model` 共享模型；测试位于 `src/test/kotlin/.../opds/`，使用 `MockWebServer` 对传输层进行隔离验证。
