- 路由集中定义在 `navigation/Screen.kt` 的 sealed class 中，带 `route` 字符串与可选 `createRoute(...)` 工厂方法，导航统一通过 `navController.navigate(Screen.Xxx.createRoute(...))` 触发。
- Feature 层 Screen Composable 通过 `viewModel(factory = XxxViewModel.Factory)` 参数注入 ViewModel，避免在 Composable 内部直接 new 实例。
- UI 颜色、圆角、阴影等视觉常量全部从 `ui/design/DesignTokens.kt` 读取，业务 Screen 不硬编码 Color/Dimen。
- 页面布局统一包裹在 `ui/design/DesignComponents.kt` 提供的 `DesignPage` 容器中，配合 `PaddedScreen(paddingValues)` 处理 Scaffold bottomBar 的 inset。
- 状态收集一律使用 `lifecycle.compose.collectAsStateWithLifecycle`，而非裸 `collectAsState`，确保生命周期感知。
- `ServiceLocator` 以 `by lazy` 延迟构造各 Repository 与数据库 DAO，并通过 companion object 的 `get(context)` 提供线程安全的单例访问。
