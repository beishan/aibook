- 应用启动链：`AiBookApplication.onCreate` 初始化 `ServiceLocator` 并在 IO 协程中调用 `serverRepository.initialize()`；`MainActivity` 通过 `setContent { AiBookTheme { AiBookApp() } }` 挂载根 Composable。
- 依赖注入采用手写单例 `ServiceLocator`（`@Volatile` + synchronized），集中构造 `AiBookDatabase`、各 Repository、`OkHttpOpdsTransport` 等，供 Feature 层通过 `viewModel(factory = XxxViewModel.Factory)` 间接获取。
- 路由由 `navigation/Screen.kt` 中的 `sealed class Screen` 统一定义，`AiBookApp.kt` 内用 `NavHost` + `composable(route, arguments)` 将 route 映射到具体 `feature/*` 下的 Screen Composable，底部 `NavigationBar` 以 `bottomTabs` 列表驱动四个主 Tab。
- 功能按 `feature/<domain>` 分目录组织，每个域包含 Screen + ViewModel（如 `shelf/ShelfScreen.kt`、`reader/ReaderScreen.kt`、`opds/OpdsScreen.kt`、`store/BookStoreScreen.kt`、`settings/SettingsScreen.kt`、`importer/LocalBookImport.kt`）。
- UI 抽象集中在 `ui/design`（`DesignComponents.kt`、`DesignTokens.kt`）和 `ui/theme/Theme.kt`，所有业务 Screen 仅消费这些共享组件与色板，不直接引用 Material3 默认样式。
- 构建产物 `build.gradle.kts` 声明 `minSdk=29`、`compileSdk/targetSdk=36`、Java/Kotlin 17，依赖四个 core 子工程（`:core:model`、`:core:network`、`:core:data`、`:core:reader`）及 Readium Kotlin Toolkit 3.1.0、Compose BOM 2025.01.01、Navigation Compose 2.8.7、Coil 3.1.0、JSoup 1.18.1。
