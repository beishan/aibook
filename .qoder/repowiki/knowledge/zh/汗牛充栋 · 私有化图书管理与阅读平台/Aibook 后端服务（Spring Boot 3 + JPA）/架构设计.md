模块遵循经典三层分层：`controller` 暴露 REST/OPDS/WebDAV/KoReader 同步等 HTTP 端点；`service` 承载业务逻辑，其中 `scraper` 子包通过 `MetadataScraper` 接口聚合 Douban/GoogleBooks/Jd/OpenLibrary 多个外部源，由 `MetadataScrapingService` 按优先级与启用状态编排调用并写入 Redis 缓存；`repository` 为 Spring Data JPA 接口，对应 `model/entity` 下的 JPA 实体（Book、User、Tag、Category、Bookmark、ReadingProgress、BookList、BookHighlight、ScanDirectory、SystemConfig）。`config` 集中装配 Jackson（禁用 Hibernate 懒加载序列化）、Redis、JPA/Hibernate 方言与连接池；`security` 通过自定义 `BasicAuthFilter`、`JwtAuthenticationFilter` 与 `WebDavMethodFilter` 组成无状态认证链，`SecurityConfig` 声明式放行 `/api/auth/**`、`/opds/**`、`/webdav/**`、`/api/sync/**` 等公开路径。全局异常由 `GlobalExceptionHandler` 统一返回 `{error,message}` JSON。入口 `AibookApplication` 开启 `@EnableCaching` 与 `@EnableScheduling`，配合 `application.yml` 中 `scanning.cron` 定时扫描本地目录。依赖方向单向：controller → service → (repository + scraper) → entity，配置类仅被启动类扫描。
